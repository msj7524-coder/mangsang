# 망상 빈자리 알림 v2

동해시 통합예약시스템(campingkorea.or.kr)의 **취소 발생(빈자리)** 을 원하는 날짜·숙소 유형 조건으로
감시하고, 조건에 맞는 취소가 생기면 **이메일**로 알려주는 대시보드입니다.
(기존 mangsang-alarm-dashboard 의 업그레이드 버전: 날짜 범위 + 숙소 유형(전체 포함) 지정 가능)

## ⚠️ 먼저 꼭 읽어주세요 — 실제 동작을 위해 필요한 작업

이 코드는 **바로 작동 가능한 완전한 뼈대(skeleton)** 입니다. 실제 화면 흐름(1단계 달력의
✅/❌ 아이콘 → 2단계 숙소유형 버튼)은 스크린샷으로 확인했지만, 정확한 HTML class/속성명까지는
모바일 환경 특성상 확인이 어려워 `monitor.js` 상단의 셀렉터 부분(`CALENDAR_ICON_SELECTOR`,
`AVAILABLE_ICON_HINT`, `ROOM_TYPE_BUTTON_SELECTOR`)은 배포 후 한 번 조정이 필요할 수 있습니다.

### 감시 전략 (효율적으로 설계됨)
1. **1단계 달력 화면만 주기적으로 확인**: 날짜별 ✅(가능)/❌(마감) 아이콘 상태를 이전 상태와 비교
2. **"전체 포함" 설정이면 여기서 끝** — 날짜가 ❌→✅로 바뀌는 순간 바로 알림
3. **특정 숙소유형만 원하는 경우에만** 상태가 바뀐 날짜에 한해 2단계까지 들어가서
   어떤 유형(전통한옥/캐빈하우스/캐라반/글램핑 등)이 열렸는지 추가 확인
   (매번 모든 날짜를 2단계까지 들어가지 않아서 훨씬 가볍고 빠릅니다)

방법(배포 후 셀렉터 검증):
1. 실제 배포된 사이트에서 브라우저 개발자도구(F12 또는 웨일 등 크로미움 브라우저의 "페이지 소스 보기")로
   달력의 ✅/❌ 아이콘 img 태그의 `alt`나 `src`(파일명), 또는 상위 td의 class를 확인합니다.
2. `monitor.js` 의 `AVAILABLE_ICON_HINT` / `FULL_ICON_HINT` 정규식을 실제 값에 맞게 수정합니다.
3. 2단계 숙소유형 버튼도 마찬가지로 비활성화된 버튼의 class(`disabled`, `off` 등)를 확인 후
   `ROOM_TYPE_BUTTON_SELECTOR` 판별 로직을 맞춰주세요.

이 작업은 제가 화면을 같이 보면서 도와드릴 수도 있으니, 실제 배포 후 캡처나 HTML을 공유해주시면
정확한 셀렉터로 바로 수정해드릴게요.

## 주요 기능

- 날짜 범위(시작일~종료일) 지정
- 감시할 시설 다중 선택 (망상오토캠핑리조트 / 망상제2오토캠핑장 / 무릉힐링캠프장 / 추암오토캠핑장)
- 숙소 유형 다중 선택 + **"전체 포함"** 옵션
- 5분 간격(설정 가능) 자동 감시 + "지금 1회 확인" 수동 버튼
- 취소(빈자리) 감지 시 이메일 알림 발송
- 최근 이력 대시보드

## 로컬 실행

```bash
npm install
cp .env.example .env   # 이메일 정보 입력
npm start
```

브라우저에서 http://localhost:3000 접속

## 카카오톡 알림 설정 ("나에게 보내기", 무료·추천)

사업자 등록 없이, 본인 카카오 계정으로 본인에게 취소 알림을 보내는 방식입니다.

1. [카카오 디벨로퍼스](https://developers.kakao.com) 접속 → 로그인 → "내 애플리케이션" → 애플리케이션 추가
2. 앱 생성 후 **요약 정보**에서 "REST API 키" 복사 → `.env` 의 `KAKAO_REST_API_KEY` 에 입력
3. 좌측 메뉴 **제품 설정 > 카카오 로그인** → 활성화 ON
4. **Redirect URI** 등록:
   - 로컬 테스트: `http://localhost:3000/auth/kakao/callback`
   - 배포 후(Render): `https://당신의주소.onrender.com/auth/kakao/callback`
   - 등록한 주소를 `.env` 의 `KAKAO_REDIRECT_URI` 에도 동일하게 입력
5. **제품 설정 > 카카오 로그인 > 동의항목**에서 "카카오톡 메시지 전송(talk_message)" 항목을 사용 설정
6. 서버 실행 후 브라우저에서 `/auth/kakao` 접속 → 카카오 로그인 동의 → 자동으로 연동 완료
   (대시보드에서도 "카카오톡 미연동 · 지금 연동하기" 링크로 바로 이동 가능)
7. 대시보드에서 알림 채널로 "카카오톡"을 선택하고 저장하면 끝!

> 참고: 액세스 토큰은 서버가 자동으로 갱신합니다. 다만 리프레시 토큰은 60일 이상
> 앱을 전혀 사용하지 않으면 만료될 수 있어요 (감시가 계속 도는 동안은 문제없습니다).

## 이메일 알림 설정 (Gmail 예시, 무료, 선택사항)

1. 구글 계정 → 보안 → 2단계 인증 켜기
2. "앱 비밀번호" 생성 (메일 항목)
3. `.env` 파일에 `GMAIL_USER`, `GMAIL_APP_PASSWORD` 입력
4. 대시보드에서 알림 채널로 "이메일"을 선택(카카오와 동시 선택도 가능)

## Render 배포

기존 사이트처럼 Render에 배포할 수 있습니다.

1. GitHub 저장소에 이 폴더를 올립니다.
2. Render → New → Web Service → 저장소 연결
3. Build Command: `npm install`
4. Start Command: `npm start`
5. Environment 탭에 `.env` 내용을 그대로 등록

**중요:** Render 무료 웹 서비스는 일정 시간 요청이 없으면 슬립 상태가 되어
백그라운드 감시(cron)가 멈출 수 있습니다. 24시간 끊김 없이 감시하려면:
- Render의 **Cron Job** 기능으로 `node monitor-once.js` 같은 스크립트를 주기 실행하거나,
- 유료 "Always On" 웹 서비스 / Background Worker 플랜을 사용하는 것을 권장합니다.

## 폴더 구조

```
server.js       - Express 서버 + API + 스케줄러
monitor.js      - 사이트 감시(스크래핑) + 상태변화 감지 + 알림 트리거
notifier.js     - 이메일 발송
config.js       - 시설/숙소유형 목록
public/         - 대시보드 화면
data/           - 감시 상태·이력 저장 (자동 생성)
```
